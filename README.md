# Асинхронная обработка вычислительных задач в C++

Программа генерирует несколько вычислительных задач, каждая из которых выполняется в отдельном потоке.  
Результаты выполнения возвращаются в основной поток через механизм `future`.  
Дополнительно реализован поток-монитор, который отслеживает завершение всех задач и сигнализирует об этом основному потоку.

## Реализовано

- Генерация **N вычислительных задач** (вычисление факториала числа).
- Выполнение каждой задачи **в отдельном потоке**.
- Использование `std::packaged_task` для упаковки вычислительной функции.
- Получение результата выполнения через `std::future`.
- Демонстрация **блокирующего поведения** `future.get()`.
- Использование `std::promise` для передачи сигнала о завершении всей системы.
- Реализация отдельного **потока-монитора**, который отслеживает завершение всех задач.
- Использование **атомарной переменной** (`std::atomic`) для подсчёта завершённых задач.
- Имитация асинхронного выполнения задач с использованием потоков `std::thread`.

## Используемые функции и классы C++

| Функция / класс | Назначение |
|-----------------|------------|
| `std::mutex`	| Защита общих данных от состояния гонки |
| `std::lock_guard` | Простая и безопасная блокировка мьютекса |
| `std::thread` | Создание и управление потоками |
| `std::packaged_task` | Обёртка для функции, позволяющая получить результат через future |
| `std::future` | Получение результата асинхронного вычисления |
| `get()` | Получение результата (блокирует поток до завершения задачи) |
| `std::promise` | Передача результата или сигнала между потоками |
| `std::atomic` | Атомарный счётчик завершённых задач |
| `fetch_add()` | Атомарное увеличение значения |
| `load()` | Чтение атомарного значения |
| `std::this_thread::sleep_for()` | Пауза для имитации ожидания |

## Принцип работы программы

1. Основной поток генерирует **N задач** вычисления факториала.
2. Каждая задача оборачивается в `std::packaged_task`.
3. Для каждой задачи:
   - создаётся `future` через `get_future()`;
   - задача передаётся в отдельный поток `std::thread`.
4. После выполнения задачи функция увеличивает атомарный счётчик `completedTasks`.
5. Основной поток получает результаты задач через `future.get()`.
   - Метод `get()` блокирует поток до завершения вычисления.
6. Параллельно работает **поток-монитор**, который:
   - отслеживает значение атомарного счётчика;
   - после завершения всех задач отправляет сигнал через `std::promise`.
7. Основной поток ожидает этот сигнал через `future`, полученный из `promise`.
